# Production artifacts


| Файл | Зачем оставлен |
|---|---|
| `policy_config.json` | Только runtime-настройки, которые читает inference: файл модели, имя политики, порог `score_threshold`, `graph_top_k` и два лимита числа кандидатов. |
| `graph_edge_model.joblib` | Обученная XGBoost-модель. Скорит пары профилей и решает, насколько связь похожа на дубль. |
| `feature_cols.json` | Точный список и порядок признаков для модели. Нужен, чтобы inference подавал признаки так же, как они подавались при обучении. |
| `historical_profile_core.parquet` | Исторические `profile_id` и `entity_id`. Нужен для назначения нового профиля к существующему клиенту и для проверки качества на размеченных пакетах. |
| `historical_profile_values.parquet` | Значения признаков исторических профилей. Нужны для расчёта совпадений и similarity-признаков между новым пакетом и историей. |
| `historical_blocking_index.parquet` | Готовый blocking index по истории. Нужен, чтобы быстро найти ограниченный список кандидатов вместо сравнения со всеми профилями. |
| `recommended_rule_names.json` | Зафиксированный список blocking-правил, которые разрешены в runtime. Нужен, чтобы inference использовал тот же набор правил, что и обученная модель. |
| `artifact_manifest.json` | Паспорт обученной модели: тип, гиперпараметры, размер train/valid и метрики выбора. Нужен для контроля версии и воспроизводимости. |
| `README_ARTIFACTS.md` | Это описание. |


## Runtime-настройки

Поля в `policy_config.json`, которые управляют объемом поиска и финальным графом:

| Параметр | Что делает |
|---|---|
| `graph_top_k` | После оценки пар моделью каждый профиль оставляет не более `K` лучших соседей. Ребро графа принимается только если два профиля взаимно выбрали друг друга. При `K=1` объединения осторожнее; увеличение `K` может повысить полноту, но увеличивает риск ошибочных цепочек объединений. |
| `max_history_candidates_per_profile` | Максимум исторических профилей-кандидатов для одного профиля входного пакета после совпадений по blocking-правилам. Ограничение защищает от очень шумного профиля, который иначе породил бы тысячи сравнений с историей. |
| `max_candidate_pairs_per_request` | Общий максимум пар `профиль входного пакета - исторический профиль`, которые переходят к расчету признаков и XGBoost за один запрос. Ограничивает время и память inference; слишком маленький лимит может отбросить правильные совпадения. |
